# Contexto da Feature: Chat NEX

## Visão Geral

Assistente de chat imersivo com dois modos operacionais:
- **Modo livre** (sem documento): responde dúvidas gerais sobre SST, NRs, EPIs, compliance e segurança do trabalho.
- **Modo grounded** (com documento): restrito ao conteúdo do documento carregado, atua como auditor forense.

## Decisões Arquiteturais

- **Abertura em Nova Janela**: botão `Assistente NEX` no `GlobalNav` abre `/chat` em popup (480x800). Evita sobreposição e conflitos de stacking context. Atalho Cmd/Ctrl+K também abre o popup.
- **Página Standalone**: rota `/chat` renderiza `ChatInterface` em layout dedicado. Quando aberta como popup, exibe botão "Fechar janela"; quando navegação direta, exibe "Voltar" para `/`.
- **Streaming SSE Nativo**: Rota `/api/chat-documento` utiliza Groq via `fetch` nativo (não SDK) para evitar APIConnectionError no Vercel; Z.AI/Ollama via `ReadableStream` manual. Resposta token-a-token.
- **Fallback bidirecional**: `AI_PROVIDER=groq` → Groq (retry 1x) → Z.AI → Groq; `AI_PROVIDER=zai` → Z.AI → Groq. Z.AI 429 aciona fallback imediato para Groq (sem retries).
- **Estados de Processamento Granulares**: O status do chat transita entre `idle`, `thinking` (análise de contexto) e `writing` (geração de resposta).
- **Persistência com TTL**: Histórico de mensagens persistido via `localStorage` com Time-To-Live de 30 dias (gerenciado por `chat-storage.ts`).
- **Composer com capacidades graduais**:
  - `Anexar`: funcional (upload + extração via `POST /api/extrair-texto`, PDF via unpdf).
  - `Modo livre`: toggle contextual quando há documento carregado.
  - `Modelo padrão` e `Voz`: expostos, porém desabilitados (`em breve`).
- **Modularização de Rota**: Builders de prompt e normalização de mensagens isolados em `prompt-builder.ts` para manter a rota enxuta (< 200 linhas).
- **Determinismo de IA**: Providers configurados com `temperature: 0`, `top_p: 1`, `seed: 42`.

## Componentes

| Componente | Responsabilidade |
|---|---|
| `ChatInterface.tsx` | Core: orquestração de mensagens, consumo de stream SSE, composer e ações (`Anexar`, modo). |
| `app/chat/page.tsx` | Página standalone `/chat` com layout dedicado (sem header principal). Popup ou navegação direta. |
| `ChatMessageBubble.tsx` | Renderização de markdown, timestamps relativos (`Intl.RelativeTimeFormat`) e toolbar de ações. Layout limpo sem avatar. |
| `ChatTypingIndicator.tsx` | Indicador verbal e icônico adaptativo às fases `thinking` (cérebro) e `writing` (lápis). Sem avatar. |
| `ChatSuggestions.tsx` | Perguntas sugeridas contextuais ao modo de operação. |

## Fluxo de Streaming

1. Usuário envia mensagem → `setChatStatus('thinking')`.
2. Timer de 1.2s inicia para mudar visualmente para `writing` caso o primeiro token demore.
3. Requisição para `POST /api/chat-documento` retorna um stream SSE.
4. Primeiro token chega → cancela timer → `setChatStatus('writing')`.
5. Tokens acumulados via `ReadableStreamDefaultReader` e inseridos na última mensagem do assistente imutavelmente via `updateLastAssistantMessage`.

## Fluxo de Anexo no Composer

1. Usuário clica em `Anexar` no composer.
2. Arquivo (`PDF`, `DOCX`, `TXT`) é enviado para `POST /api/extrair-texto`.
3. `documentContext` e `documentName` são atualizados no `ChatContext`.
4. Modo grounded é ativado por padrão após extração bem-sucedida.
5. Usuário pode alternar para `Modo livre` (sem perder o contexto carregado).

## Utilidades e Libs

- `chat-storage.ts`: Abstração de persistência com envelope de expiração (30 dias).
- `chat-utils.ts`: Formatação de tempo relativo e manipulação imutável de arrays de mensagens.
- `react-markdown` + `remark-gfm`: Renderização técnica de respostas.
- `framer-motion`: Animações de entrada e transições de estado (respeitando `useReducedMotion`).
