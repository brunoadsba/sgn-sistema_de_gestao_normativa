# Contexto da Feature: Chat NEX

## Visão Geral

Assistente de chat imersivo com dois modos operacionais:
- **Modo livre** (sem documento): responde dúvidas gerais sobre SST, NRs, EPIs, compliance e segurança do trabalho.
- **Modo grounded** (com documento): restrito ao conteúdo do documento carregado, atua como auditor forense.

A transição entre modos é automática: ao carregar um documento, o chat reinicia e muda para grounded.

## Decisões Arquiteturais

- **Modal Central Imersivo**: Chat acessível via Header (`GlobalNav.tsx`), abandonando o painel lateral/FAB para um modal central com backdrop blur e foco total na conversa.
- **Streaming SSE Nativo**: Rota `/api/chat-documento` utiliza `stream: true` (Groq SDK) com fallback manual via `ReadableStream` para outros providers, garantindo resposta token-a-token.
- **Estados de Processamento Granulares**: O status do chat transita entre `idle`, `thinking` (análise de contexto) e `writing` (geração de resposta).
- **Persistência com TTL**: Histórico de mensagens persistido via `localStorage` com Time-To-Live de 30 dias (gerenciado por `chat-storage.ts`).
- **Modularização de Rota**: Builders de prompt e normalização de mensagens isolados em `prompt-builder.ts` para manter a rota enxuta (< 200 linhas).
- **Determinismo de IA**: Providers configurados com `temperature: 0`, `top_p: 1`, `seed: 42`.

## Componentes

| Componente | Responsabilidade |
|---|---|
| `ChatInterface.tsx` | Core: orquestração de mensagens, consumo de stream SSE, scroll âncora e estados de UI. |
| `ChatMessageBubble.tsx` | Renderização de markdown, timestamps relativos (`Intl.RelativeTimeFormat`) e toolbar de ações. |
| `ChatTypingIndicator.tsx` | Indicador verbal e icônico adaptativo às fases `thinking` (cérebro) e `writing` (lápis). |
| `ChatSuggestions.tsx` | Perguntas sugeridas contextuais ao modo de operação. |

## Fluxo de Streaming

1. Usuário envia mensagem → `setChatStatus('thinking')`.
2. Timer de 1.2s inicia para mudar visualmente para `writing` caso o primeiro token demore.
3. Requisição para `POST /api/chat-documento` retorna um stream SSE.
4. Primeiro token chega → cancela timer → `setChatStatus('writing')`.
5. Tokens acumulados via `ReadableStreamDefaultReader` e inseridos na última mensagem do assistente imutavelmente via `updateLastAssistantMessage`.

## Utilidades e Libs

- `chat-storage.ts`: Abstração de persistência com envelope de expiração (30 dias).
- `chat-utils.ts`: Formatação de tempo relativo e manipulação imutável de arrays de mensagens.
- `react-markdown` + `remark-gfm`: Renderização técnica de respostas.
- `framer-motion`: Animações de entrada e transições de estado (respeitando `useReducedMotion`).
